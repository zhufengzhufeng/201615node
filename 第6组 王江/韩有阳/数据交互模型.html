客户端和服务器端的交互模型

问题：我们平时上网，从打开浏览器到看到页面，中间都经历了什么
客户端：发送请求的是客户端
服务器端：接收请求把内容返回的是服务器端
例如我们平时访问百度，我们电脑的浏览器就是客户端，百度公司的服务器就是服务器端

开通博客

  • 买一台服务器（阿里云服务器/虚拟主机）、给服务器连接上网络，此时服务器就有一个外网的IP地址，（通过一个IP地址，可以访问到这台服务器）也有人把它叫做主机地址
  • 把自己的项目 源代码上传到服务器上。（FTP上传工具）
  • 购买一个域名。（因为域名好几亿，IP地址都是数字，不方便记忆），万维网
    =>域名解析（DNS解析->把解析的记录存放在DNS服务器上的）=> 把域名和外网的IP地址关联在一起。
    => DNS服务器也叫作域名解析服务器，属于国际互联网阻止统一管理的。
  • 我们还需要再服务器上创建服务,让服务器负责指定的项目文件的管理，区分不同的服务需要使用端口号来区分（端口号取值范围：0~65535之间）；

换个案例理解：开超市

  • 租店面（购买服务器），店面有一个具体的地址（主机地址）；
  • 把需要卖的物品摆放在店里（FTP把资源文件上传服务器）；
  • 给店面起一个名字：XXX超市，（相当于购买域名）；
  • 把店名和具体地址印在宣传彩页上，让别人知道具体的地址；（DNS解析）
  • 以上事情都完成，店也不能开业，还缺售货员（需要多个售货员）；如果是多个售货员，我们需要指定某一个售货员负责哪部分；我们还需要区分多个售货员，我们需要给售货员增加不同的编号；

前后端交互模型

  1. 客户端到DNS服务器上通过域名找到主机地址
  2. 通过主机IP找到服务器
  3. 通过端口号找到对应的服务
  4. 找到具体想要的资源文件
  5. 服务器把需要的资源文件中的”源代码”返回给客户的浏览器
  6. 浏览器对源代码进行渲染和解析.

PS：一个完整的前后端（客户端和服务器端）交互就是由请求+响应共同组成的
1+2+3 => 请求阶段 Request
4+5 => 响应阶段 Response

细节

客户端发送一个请求，服务器接收到这个请求，然后把需要的内让以源代码的方式返回；再这个过程中服务器给客户端传递内容了，真实的项目中，不仅仅服务器端会把内容传递给客户端，有些时候客户端还要把一些内容传递给服务器；例如买东西，客户需要给钱，售货员需要给物品。
一个完整的URL地址
http://www.zhufengpeixun.cn:80/index.html?name=zxt&age28#happy

  1. 第一部分：传输协议
    ps: 就是用来实现客户端和服务器之间的内容传输的。

    • http: —> 超文本协议,也就是不仅仅可以传递文本内容，还可以传递二进制编码或者文件流（图片，音视频）等内容
    • https： –> 更加安全的http, 全称HTTP SSL，给传递的内容进行了加密，一般需要金钱支付的网站都是HTTPS协议。
    • FTP：传输大的内容，（例如：把项目资源文件上传到服务器…）
    • …

  2. 第二部分：域名
    相当于给不好以及的主机IP地址起一个好记忆的名字

    • 一级域名：www.qq.com
    • 二级域名：sports.qq.com
    • 三级域名：kbs.sports.qq.com
    • …

  3. 第三部分：端口号
    端口号作用：用来区分同一台服务器上的不同服务的，端口号取值范围:0~65535之间

    • HTTP默认端口号80，只要使用的HTTO协议，再我们没有指定端口号的情况下，浏览器默认给加上的端口号就是80；
    • HTPPS默认端口号是443；
    • FTP默认端口号是21；

  4. 第四部分：客户端请求资源文件的路径名称
    例如：/index.html /js/index.js…….客户端指定请求的路径名称，服务器根据这个信息把具体的文件中的源代码读取到然后给客户端返回即可

当我们没有指定的情况，大部分默认请求的都是项目根目录下的index.html,也就是默认会增加/index.html (当然这个默认的值是服务器可以修改配置的)；

  1. 第五部分：？号传参
    ？name = zxt&age = 28 URL地址栏问好传参，通过这种方法，客户端可以把一些信息传递给服务器，再当前的案例中，客户端就把name=zxt以及age=28传递给服务器了

一个URL中只能出现一个问号，多组内容之间用&隔开

再真实项目中，我们有一个列表页面，列表页面中有很多条内容，点击每一条内容进入到详情页面，详情页面展示的是不一样的内容；

  • 首先我们需要知道的是，详情页面是同一个HTML页面
  • 如何在详情页面中展示不同的内容呐？ 通过的就是问号传递参数，点击列表页面中的某一条记录，跳转到详情页的时候，我们在URL的末尾用？传递点值，点击不同的的数据，传递的值不一样。
  • 进入到详情页，我们获取到URL中问号传递的参数值。

  • 通过不同的参数值获取到不同的数据来展示

    1. 第六部分 哈希值(HASH) #happy
      锚点定位：定位到当前页面中 ID为 HASH值的盒子的位置
      通过HASH值内容传递给服务器（等同于问好传递参数的作用），一般都用问号传递参数，很少用这个在单页面应用中，通过HASH值的不同，实现前端路由切换。

客户端与服务器的交互

  • Request:请求阶段，客户端向服务器发送请求。
  • Response:响应阶段，服务器把请求资源文件的”源代码”返回给客户端。

在这个交互过程中，客户端可以把内容传递给服务器，服务器也可以把内容返回给客户端，客户端和服务器端的内容传输基本上都是交给HTTP（HTTPS）传输协议来完成的。

客户端和服务器端相互传输的内容统称为：HTTP报文。

客户端把内容传递给服务器和服务器把内容传输给客户端都是有很多方式的。

NODE

node不是后台开发语言，我们了解的JAVA、PHP、C#….NET(dot net)….才是后台开发语言,NODE仅仅是一个工具，一个基于V8引擎来渲染和JS的平台和工具（有点类似浏览器）

真实的开发中，我们一般都把NODE安装太服务器端，这样我们就可以在服务器端编写一些JS代码来处理服务器端的业务逻辑了=>JS不仅仅可以在客户端浏览器中运行处理客户端业务逻辑（前端开发语言），也可以在服务器端的NODE环境下运行处理服务器端业务逻辑（后台开发语言），所以JS是全栈开发语言

node如何去执行JS代码

  • 直接利用WB：再JS的空白处，右键，找到 Run xxx.js 就是再WB中调取， NODE环境下执行JS代码
  • NODE的REPL命令

NODE的优势

  1. 快：基于V8引擎渲染和解析的。
  2. NODE是单线程基于事件驱动的异步操作。
  3. NODE提供了无阻塞的I/O操作

I/O：input /output 对文件的增删改查操作。

  1. JS再客户端浏览器中运行的时候是否可以对客户端本地的文件进行I/O操作？
    不可能，因为我们要保证客户端的安全，JS做为前端开发语言是不可以操作客户端本地文件的，但是浏览器也提供了部分需要用户自主操作的功能，例如：上传图片
  2. JS再服务器端NODE环境下运行的时候是否可以对服务器上额文件进行I/O操作？ 可以的

使用npm安装模块

  1. 第一种安装在全局(global)
    npm install xxx(模块名字) -g
    npm unistall xxx(模块名字) -g 卸载已经安装在全局的模块

  2. 安装在当前项目下
    npm install xxx(模块名字)
    npm uninstall xxx(模块名字) 卸载已经安装在当前项目下的模块

把模块安装在全局出现的问题

  • 把模块安装在全局，以后不管在哪一个项目目录下，我们都可以”通过使用命令行的方式”操作进行操作了，但是不是能通过代码导入这个模块使用。
  • 安装在全局会导致版本号冲突，例如：我之前安装的是less是2.7版本，以前的项目是使用这个版本开发的，后来less升级了，升级到3.1，新版本和之前的版本不太一样了，如果全局安装最新的，原来的项目就没办法使用LESS了。
  • 综合以上两点问题，我们一般再开发的时候，我们都把第三方模块安装在当前项目中

把模块安装在当前的项目中也有问题：

  • 不能像全局一样使用命令了，但是可以使用js导入了。
  • 在当前的项目中会有一个 node madules文件夹，里面存储的都是我们当前项目所需要使用的模块（这个文件夹很大的，传递给其他人非常的不好处理）
  • 我们把一些第三方模块安装在A项目下，B项目不能直接的导入使用，如果需要的话还需要再B模块中重新的安装一遍
